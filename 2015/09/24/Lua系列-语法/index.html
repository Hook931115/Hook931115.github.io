Lua系列-语法

1.命名规范

Lua中的命名规范基本上和C语言是一样的。

Lua中的标示符可以由任意的字母组合、数字和下划线构成，但不能以数字开头，并且避免下划线开头和后面是几个大写字母的命名方式，下面是几个正确的命名方式：

i j var abcdefg ab_cd1234

不要使用Lua中的关键字，Lua中的关键字很少，只有21个，命名变量时，不能说使用相同的名称：

and      break     do     else        elseif
end      false     for    function    if
in       local     nil    not         or
repeat   return    then   true        until
while

Lua对大小写敏感，大写和小写是两个不同的单词

2.类型和值

Lua是一种动态类型的语言，在语言中没有类型定义的语法，每个值本身都包含了类型信息，对于变量来说，他的值的类型是可以随意变换的。Lua中又8种基础类型：

  • nil：空
  • boolean：布尔
  • number：数字
  • string：字符串
  • funciton：函数
  • table：表
  • userdata：自定义的数据类型
  • thread：线程

函数type可以根据传入的参数返回该参数的类型名称。

print(type(var))
var = 20 
print(type(var))
var = "hello world"
print(type(var))
var = print
print(type(var))
var = {}
print(type(var))

2.1 nil

在Lua中nil表示的是一种类型，它只包含一个值nil。在变量未赋值之前都是nil，Lua中nil主要用于表示变量没有赋值，用来区分其他变量，当一个全局变量为nil时表示删除这个变量。

2.2 boolean

boolean有两个值：true 和 false。和C++中bool是一样的。在lua中只有false和nil表示假，其他都表示真。（特别注意数字 0）

var = nil
if var then
    print("var is true")
else
    print("var is false")
end
  • 当var没有被赋值时，输出为var is false
  • 当var赋值为false时，输出为var is false
  • 当var赋值为true时，输出为var is true
  • 当var赋值为0时，输出为var is true（不要和C/C++弄混了）
  • 当var被赋值为“”时，输出为var is true

2.3 number

在lua中number表示实数。不管是浮点数还是整型数字都是number类型。有人可能会担心浮点数的效率，其实现在的CPU大部分都有单独的浮点数计算协处理器，对于浮点数的计算完全不是问题。如果真的担心的话可以在编译lua时将number类型的类型改变成自己需要的类型（luaconf.h）。lua中的number既可以使用普通的写法，也可以使用科学计数法。

a = 10; b = 50.2; c = 5e20

2.4 string

lua中的字符串表示的是一个字符序列，里面可以包含任何的字符，包括数字0（C/C++中字符串的终止字符）。并且Lua中字符串的长度是不限制的，你皆可以时空字符串也可以时整整一本书，lua对字符串的处理时很卡的，一般定义一个字符串是这样的

str = "I'm a stirng"

在Lua中，单引号和双引号都尅作为字符串的标识，建议统一使用一种。如果你的字符串有多种格式的画可以使用下面的方式来定义字符串：

long_str = [[I'm a 
            "long",
            long
            str]]

如果你的字符串中又多个连在一起的方括号时，可以再两个方括号之间添加添加多了=，比如：

long_str = [===[[[I'm a 
            "long",
            long
            str
            ]]]===]

当需要连接两个字符串时，你可以使用字符串连接操作符..，类似于C语言中的strcat，比如：

print("Hello".."world")

当我们需要获取字符串的长度时，使用#能获取字符串的总长度，比如：

a = "Hello world"
print(#a)

2.5 table

Lua中的table类似于C++中的map，但是相对于map的繁琐，table大大简化了使用过程中的操作。比如我们要再C++中声明一个map，我们需要这样来定义：

std::map<std::string,std::string> MyMap;
MyMap["a"] = "test";
MyMap.insert(std::map<std::string,std::string>::value_type("b","test2"));

但是在Lua中我们只需要这样声明就够了：

t = {a = "test" , b = "test2"}
